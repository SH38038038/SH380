# 1\. 이론

## 스위치(Switch)

-   **역할**: LAN(Local Area Network) 내 장치들을 연결하고 데이터 패킷을 목적지 MAC 주소 기반으로 전달
-   **동작 계층**: OSI 2계층(Data Link Layer)
-   **특징**
    -   MAC 주소 학습 및 전송
    -   같은 네트워크 대역 내 장치 연결
    -   브로드캐스트 도메인은 그대로 유지

## 라우터(Router)

-   **역할**: 서로 다른 네트워크 간 데이터 전달 (LAN ↔ WAN, LAN ↔ LAN)
-   **동작 계층**: OSI 3계층(Network Layer)
-   **특징**
    -   IP 주소 기반 라우팅
    -   서로 다른 네트워크 대역 연결
    -   NAT, DHCP, 방화벽 등 네트워크 서비스 제공 가능

  
  

# 2\. 비교 정리

| 구분 | 스위치(Switch) | 라우터(Router) |
| --- | --- | --- |
| OSI 계층 | 2계층 (데이터 링크) | 3계층 (네트워크) |
| 기준 | MAC 주소 | IP 주소 |
| 역할 | 같은 네트워크 내부 장치 연결 | 다른 네트워크 간 연결 |
| 주요 기능 | 프레임 전달, 충돌 도메인 분리 | 라우팅, NAT, 보안 기능 |
| 사용 예 | 회사 내부 PC ↔ 프린터 연결 | 사내망 ↔ 인터넷 연결 |

  
  

# 3\. 활용

-   **스위치**
    -   사무실 내부 PC, 프린터, 서버 연결
    -   대규모 LAN 트래픽 관리
-   **라우터**
    -   집에서 공유기(라우터)로 인터넷 연결
    -   기업 네트워크와 인터넷망 연결
    -   데이터 센터에서 다중 네트워크 간 트래픽 제어

  
  

# 4\. 심화 (네트워크 성능 관점)

-   **스위치**
    -   고속 포트(1Gbps, 10Gbps)로 LAN 성능 향상
    -   VLAN 지원 → 네트워크 논리적 분리
-   **라우터**
    -   고급 라우팅 프로토콜 (OSPF, BGP) 지원
    -   QoS(Quality of Service) → 특정 트래픽 우선순위 보장
    -   방화벽 역할 병행 가능

# 5\. 면접 연습
​
-   스위치: **같은 네트워크 내 통신 (MAC 기반)**
-   라우터: **네트워크 간 통신 (IP 기반)**
-   OSI 계층 구분 필수 (2계층 vs 3계층)
-   라우터는 NAT/DHCP 같은 **부가 기능**까지 가능
-   실제 사용 예시를 함께 설명하면 가산점
​
  
  
​
# 6\. 실제 면접 질문 & 답변
​
Q1. 스위치와 라우터의 가장 큰 차이는? 
A: 스위치는 같은 네트워크 내에서 MAC 주소 기반 데이터 전달, 라우터는 서로 다른 네트워크 간 IP 기반 데이터 전달을 합니다. 

Q2. 집에서 인터넷을 쓸 때 스위치와 라우터 중 무엇이 필요할까요? 
A: 라우터가 필요합니다. ISP(통신사)와 집 내부 네트워크를 연결하고, NAT/DHCP 등을 통해 여러 기기에 인터넷을 할당합니다. 

Q3. VLAN을 지원하는 장비는 스위치일까요, 라우터일까요? 
A: 스위치입니다. VLAN은 하나의 스위치를 여러 논리적 네트워크로 분할하여 관리할 수 있습니다. 

Q4. 기업 네트워크에서 스위치와 라우터는 어떻게 배치되나요? 
A: 내부 장치 연결은 스위치가 담당, 외부 인터넷이나 다른 네트워크 연결은 라우터가 담당합니다.
​
  
  
​
# 7\. 마무리
​
-   스위치: **LAN 내부 연결 담당 (MAC 기반, 2계층)**
-   라우터: **네트워크 간 연결 담당 (IP 기반, 3계층)**
-   면접에서는 **계층/주소 기준/활용 사례** 3가지를 묶어 설명하면 완벽
